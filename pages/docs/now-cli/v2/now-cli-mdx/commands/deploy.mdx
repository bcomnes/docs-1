## Deploy

The `now deploy` command, commonly shortened to just `now`, is used to deploy projects and can be used from either the root of the project directory, or by providing a path to it.

### Basic Usage

<Snippet dark text="now" />
<Caption>Using the <InlineCode>now</InlineCode> command from the root of a project directory.</Caption>

### Extended Usage

<Snippet dark text="now [path-to-project]" />
<Caption>Using the <InlineCode>now</InlineCode> command and supplying a path to the root directory of the project.</Caption>

### Options

The following options can be passed when using the `now` command:

- `--debug`
- `--force`
- `--help`

### Unique Options

These are options that only apply to the `now deploy` command, therefore, more information is provided.

- `--build-env` - provides the [build step](/docs/v2/build-step) with an environment variable
- `--env` - provides the [run time](/docs/v2/serverless-functions#env-and-secrets) with an environment variable
- `--name` - provides a project name
- `--prod` - generates a deployment for a production domain
- `--public` - ensures the source code is accessible at the `/_src` paths
- `--regions` - specifies which regions the [Serverless Functions](/docs/v2/serverless-functions) should run in
- `--version` - returns informationg about the [Now CLI](/download) platform version

For more information on options and their usage, refer to the [options section](#options).
