## Certs

The `now certs` command is used to manage certificates for domains, providing functionality to list, issue and remove them.

<Note type="warning">
  This command is intended for <b>advanced use only</b>. By default, ZEIT Now
  manages your certificates automatically.
</Note>

### Basic Usage

<Snippet dark text="now certs ls" />
<Caption>Using the <InlineCode>now certs</InlineCode> command to list all certificates under an account.</Caption>

### Extended Usage

<Snippet dark text="now certs issue [domain1, domain2, domain3]" />
<Caption>Using the <InlineCode>now certs</InlineCode> command to issue certificates for multiple domains.</Caption>

<Snippet dark text="now certs rm [certificate-id]" />
<Caption>Using the <InlineCode>now certs</InlineCode> command to remove a certificate by ID.</Caption>

### Options Available

The following options can be passed when using the `now certs` command:

- `--debug`
- `--env`
- `--force`
- `--global-config`
- `--help`
- `--local-config`
- `--scope`
- `--token`

### Unique Options

These are the options that only apply to this particular Now CLI [command](#commands), therefore, more information is provided.

- `--challenge-only` - only show the challenges needed to issue a certificate
- `--crt [file-path]` - upload a certificate file for the domain
- `--key [key-path]` - upload a certificate key file for the domain
- `--ca [ca-path]` - upload a CA certificate chain file for the domain

For more information on options and their usage, refer to the [options section](#options).
